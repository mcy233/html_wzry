# 选手养成与转会招募系统策划案

> 版本：v1.1 | 日期：2026-04-17
> 参考：《英雄联盟电竞经理》选手系统 + 适配本项目（HTML单机小游戏）特点

---

## 一、设计目标

1. **长线养成深度**：让选手不再是固定数值的"静态卡"，而是可以持续成长、有收集欲望的"动态资产"
2. **经济闭环**：训练→经验→升级→战力↑ / 赛事奖励→金币→签约→新选手→阵容深度↑
3. **适配单机HTML**：不做真实付费，用游戏内货币和游玩时间替代氪金；不做服务器竞拍，改为单机随机池

---

## 二、参考分析：《英雄联盟电竞经理》

| 系统 | 电竞经理做法 | 我们的适配 |
|------|-------------|-----------|
| **选手品质** | N / R / SR / SSR 四档 | R（蓝）/ SR（紫）/ SSR（金）三档 |
| **获取方式** | 金券抽卡(氪金)、友情券抽卡、碎片合成、转会竞拍 | 签约券抽卡(赛事/任务奖励)、碎片合成、转会市场(NPC刷新) |
| **抽卡保底** | 50抽保底SSR | 30抽保底SSR（单机节奏更快） |
| **升级** | 资金 + 训练数据 → 属性提升，每10级突破 | 金币 + 训练经验 → 属性提升，每10级突破 |
| **升星** | 碎片 + 鉴定书 + 资金 → 大幅属性/技能提升 | 碎片 + 金币 → 星级↑，属性大幅提升 |
| **升品** | R→SR→SSR（碎片+鉴定书） | R→SR→SSR（大量碎片+金币） |
| **羁绊** | 战队羁绊 + 选手羁绊，上阵/收集双重加成 | 战队羁绊（同队5人上阵）+ 双人组合羁绊 |
| **图鉴** | 收集激活，累计图鉴值获奖励 | 选手图鉴，收集解锁全队永久加成 |
| **训练** | 消耗训练数据，选手升级 | 训练小游戏 → 获得训练经验 → 选手升级消耗 |

## 三、选手卡系统

### 3.1 选手卡品质

| 品质 | 颜色 | 基础评分范围 | 初始技能数 | 抽取概率 |
|------|------|-------------|-----------|---------|
| **R**（稀有） | 蓝色 | 65~75 | 0 | 70% |
| **SR**（超稀有） | 紫色 | 76~85 | 1 | 25% |
| **SSR**（传说） | 金色 | 86~95 | 2 | 5% |

- 每位KPL真实选手在数据中预设一个**基础品质**（对应真实实力）
- 同一位选手可以同时存在不同赛季版本的卡（如 "2027春 暖阳" vs "2027夏 暖阳"），后续可扩展

### 3.2 选手卡属性结构

沿用现有六维属性，新增成长相关字段：

```
{
  id: '暖阳',
  name: '林恒',
  role: '打野',
  rarity: 'SSR',        // 品质
  rating: 90,            // 综合评分（随等级/星级动态计算）
  level: 1,              // 等级（1~60）
  star: 1,               // 星级（1~5，之后红1~红5）
  exp: 0,                // 当前经验值
  fragments: 0,          // 持有碎片数
  stats: { 操作: 92, 意识: 93, 对线: 82, 配合: 88, 抗压: 87, 心态: 89 },
  // 技能（SSR有2个，SR有1个，R无）
  skills: [
    { name: '丛林霸主', desc: '打野效率+15%，前期节奏更强', effect: { jungle: 0.15 } },
    { name: '团战指挥', desc: '团战阶段全队意识+5', effect: { teamFight: 5 } },
  ],
  condition: 80,         // 状态（已有）
}
```

### 3.3 等级系统

| 等级段 | 等级范围 | 单级所需经验 | 升级效果 |
|--------|---------|-------------|---------|
| 初级 | 1~10 | 100~200 | 每级随机1项属性+1 |
| 中级 | 11~20 | 250~400 | 每级随机1项属性+1 |
| 高级 | 21~40 | 500~800 | 每级随机1项属性+1 |
| 大师 | 41~60 | 1000~1500 | 每级随机1项属性+1 |

**突破机制**（每10级一次）：
- 10级突破：所有属性+2
- 20级突破：所有属性+2，解锁自由分配点 ×3
- 30级突破：所有属性+3
- 40级突破：所有属性+3，解锁自由分配点 ×5
- 50级突破：所有属性+4，解锁第二技能（如果品质允许）
- 60级（满级）：额外获得专属称号加成

升级消耗：**训练经验** + **金币**（金币消耗 = 等级 × 10）

### 3.4 星级系统

| 星级 | 碎片消耗 | 金币消耗 | 效果 |
|------|---------|---------|------|
| 1★ → 2★ | 5 | 200 | 所有属性+2 |
| 2★ → 3★ | 10 | 500 | 所有属性+3，技能等级+1 |
| 3★ → 4★ | 20 | 1000 | 所有属性+4，技能等级+1 |
| 4★ → 5★ | 40 | 2000 | 所有属性+5，解锁3点自由分配 |
| 5★ → 红1★ | 60 | 3000 | 所有属性+5，技能等级+1 |
| 红1★→红5★ | 每级80 | 每级5000 | 每级所有属性+3 |

碎片获取途径：
- 签约（抽卡）时获得重复选手 → 自动转为碎片（R=2片，SR=5片，SSR=10片）
- 赛事日历中特定焦点赛扫荡 → 获得特定选手碎片
- 转会市场购买碎片
- 分解不需要的选手卡 → 获得碎片

### 3.5 升品系统

| 升品路径 | 条件 | 碎片消耗 | 金币消耗 |
|---------|------|---------|---------|
| R → SR | 5★ | 50碎片 | 3000 |
| SR → SSR | 5★ | 100碎片 | 8000 |

升品后：
- 品质框变色
- 初始评分按新品质基线重算（但不会低于升品前）
- 解锁更多技能槽

> ⚠️ 升品后的选手仍弱于同品质原生卡约 5~10%，引导玩家追求原生高品质卡

## 四、签约（抽卡）系统

### 4.1 签约券

| 券种 | 获取方式 | 抽取池 |
|------|---------|-------|
| **金色签约券** | 赛事奖励、成就、赛季通行证 | 全选手池（含SSR） |
| **蓝色签约券** | 每日任务、训练小游戏MVP、城市探索 | 基础池（R为主，小概率SR） |

### 4.2 抽卡机制

- **单抽**：1张签约券
- **十连**：10张签约券，保底至少1张SR
- **大保底**：30次金色签约内必出SSR（计数在banner内累积，跨banner不重置）
- **心愿单**：可设置3位选手，抽到SSR时有30%/60%/90%概率为心愿选手

### 4.3 签约池（Banner）

每个赛季阶段更新一个签约池：
- **春季赛池**：春季赛表现优异的选手SSR概率UP
- **夏季赛池**：夏季赛版本选手
- **限定池**：特殊活动（如全明星、总决赛纪念）限定选手

## 五、转会市场

### 5.1 个人转会市场

- 每完成一个赛季阶段刷新一次（5~8名选手）
- 展示选手卡面、品质、属性、价格（签约金/金币）
- 玩家可直接购买

| 品质 | 价格范围 |
|------|---------|
| R | 200~500 金币 |
| SR | 800~2000 金币 |
| SSR | 3000~8000 金币 |

### 5.2 碎片商店

- 用通用碎片（分解选手获得）兑换指定选手碎片
- 汇率：3 通用碎片 = 1 指定碎片（R）/ 5 = 1（SR）/ 10 = 1（SSR）

### 5.3 选手解约

- 不需要的选手可解约，获得：
  - 签约金（按品质和等级折算）
  - 通用碎片（R=2, SR=5, SSR=10）
- 解约前提醒激活图鉴

## 六、训练系统改造

### 6.1 核心变更

**现在**：训练小游戏 → 直接提升属性
**改为**：训练小游戏 → 获得 **训练经验** → 用经验给选手升级

### 6.2 训练项目

| 项目 | 小游戏类型 | 获得经验 | 消耗金币 |
|------|-----------|---------|---------|
| ⚡ 手速挑战 | 限时连击 | 评分×基础50 | 30 |
| 🧠 记忆矩阵 | 记忆翻牌 | 评分×基础50 | 30 |
| 🎯 节奏打击 | 节奏QTE | 评分×基础50 | 30 |
| 🤝 团队默契 | 配对连线 | 评分×基础50 | 30 |
| 🏟️ 模拟对战 | 策略选择 | 评分×基础80 | 80 |

评分等级与经验倍率：
- S级（≥85%）：×2.0 → 获得 100~160 经验
- A级（≥65%）：×1.5 → 获得 75~120 经验
- B级（≥45%）：×1.0 → 获得 50~80 经验
- C级（<45%）：×0.5 → 获得 25~40 经验

### 6.3 训练经验分配

训练获得的经验进入 **经验池**，玩家在选手管理界面自行分配给指定选手：
- 这样玩家可以集中培养核心选手
- 也可以均匀分配给全队

### 6.4 其他经验获取途径

| 途径 | 经验量 |
|------|-------|
| 赢得常规赛 | 200 |
| 输掉常规赛 | 50 |
| 赢得季后赛 | 400 |
| 赢得总决赛 | 800 |
| 完成每日任务 | 30 |
| 城市探索事件 | 20~80（随机） |
| 联赛焦点赛扫荡 | 50 + 碎片 |

## 八、战力系统

### 8.1 设计理念

将选手的品质、等级、星级、六维属性、技能、状态等所有成长维度，统一量化为一个**战力值（CP, Combat Power）**。最终在比赛前，玩家可以一目了然地看到双方团队战力对比，心中有数。

```
选手个人战力  →  五人战力总和  →  + 羁绊加成  →  + Buff加成  →  团队总战力
                                   + 图鉴加成
```

### 8.2 选手个人战力计算

```
个人战力 = 基础战力 + 等级战力 + 星级战力 + 品质战力 + 技能战力 + 状态修正
```

| 分项 | 公式 | 说明 |
|------|------|------|
| **基础战力** | `(操作+意识+对线+配合+抗压+心态) × 5` | 六维之和 × 5，满分约 99×6×5 = 2970 |
| **等级战力** | `等级 × 15` | 满级60 = 900 |
| **星级战力** | `星级阶段值` (见下表) | 1★=0, 2★=80, 3★=200, 4★=360, 5★=560, 红1★=800, 红2★=1060 ... 红5★=1880 |
| **品质战力** | R=0, SR=200, SSR=500 | 原生品质加成 |
| **技能战力** | `每个技能 × 100` | SSR双技能 = 200, SR单技能 = 100 |
| **状态修正** | `(状态 - 80) × 5` | 状态80为基准不修正，100=+100, 50=−150 |

**星级战力详表**：

| 星级 | 战力值 | 累计增量 |
|------|--------|---------|
| 1★ | 0 | — |
| 2★ | 80 | +80 |
| 3★ | 200 | +120 |
| 4★ | 360 | +160 |
| 5★ | 560 | +200 |
| 红1★ | 800 | +240 |
| 红2★ | 1060 | +260 |
| 红3★ | 1340 | +280 |
| 红4★ | 1640 | +300 |
| 红5★ | 1880 | +240 |

**示例**：1级1★ SSR暖阳（六维总和531，无技能升级，状态80）

```
基础 = 531 × 5       = 2655
等级 = 1 × 15        = 15
星级 = 0              = 0
品质 = SSR            = 500
技能 = 2个            = 200
状态 = (80-80)×5     = 0
─────────────────────
个人战力             = 3370
```

**示例**：60级红5★ SSR暖阳（六维总和约630，状态90）

```
基础 = 630 × 5       = 3150
等级 = 60 × 15       = 900
星级 = 红5★          = 1880
品质 = SSR            = 500
技能 = 2个            = 200
状态 = (90-80)×5     = 50
─────────────────────
个人战力             = 6680
```

### 8.3 团队战力计算

```
团队战力 = Σ(五名首发个人战力) × (1 + 羁绊加成率 + 图鉴加成率 + Buff加成率)
```

| 加成来源 | 加成率 | 说明 |
|---------|--------|------|
| **战队羁绊** | +8%~12% | 五人同队触发，各战队不同 |
| **选手羁绊** | 每组+2%~4% | 可叠加，最多同时激活3组 |
| **图鉴收集加成** | +1%~5% | 图鉴值达到25/50/80/全收集阈值时累加 |
| **战队Buff** | +1%~5% | 战队自带Buff（如"京城底蕴"） |
| **士气加成** | −8%~+5% | 士气>80 → +5%, 士气<40 → −8% |
| **教练祝福** | +3% | 主角教练固定加成 |

**示例**：五名WB首发（均为中等养成），个人战力之和 = 20000

```
战队羁绊(京城铁军)  = +10%
选手羁绊(双子星)    = +3%
图鉴加成(图鉴值25)  = +1%
战队Buff            = +2%
士气85              = +5%
教练祝福            = +3%
─────────────────────
总加成率 = 24%
团队战力 = 20000 × 1.24 = 24,800
```

### 8.4 战力展示位置

| 位置 | 展示内容 |
|------|---------|
| **HomeScene 顶栏** | 我方团队总战力，旁边显示加成明细图标 |
| **赛事日历·对战详情** | 我方战力 vs 对手战力（双方数值 + 柱状对比条） |
| **BP场景·顶部** | 实时战力对比（随BP选人动态变化） |
| **选手管理·卡片** | 每位选手个人战力值 |
| **选手详情** | 战力构成饼图（基础/等级/星级/品质/技能/状态占比） |
| **阵容管理** | 当前阵容战力 + 拖拽换人后的战力变化预览 |

### 8.5 对手战力计算

对手（AI控制的NPC战队）的战力按以下规则计算：

- **基础**：取该战队 teams.js 中选手的原始 stats 数据
- **成长模拟**：根据赛季进度，AI战队也有"虚拟养成"
  - 常规赛第1轮：AI选手等级约 5~10，1~2★
  - 常规赛第3轮：AI选手等级约 15~25，2~3★
  - 季后赛：AI选手等级约 30~40，3~4★
  - 总决赛：AI选手等级约 45~55，4~5★
- **难度系数**：根据对手战队原始实力 rating 和组别（S>A>B），在基础上浮动 ±10%

这确保了：
1. 赛季初期玩家有优势窗口期
2. 越往后打，AI越强，需要持续养成
3. 强队始终有压力感，弱队可以"养成翻盘"

## 九、快速战斗系统

### 9.1 设计目标

- 提供一个**跳过BP和5节点策略对决**的快速通道，方便玩家快速推进赛程、测试阵容
- 结果主要由**双方团队战力对比**决定，加入随机波动保持不确定性
- 配合简洁的**战斗播报动画**（3~8秒），保留最低限度的仪式感

### 9.2 胜率公式

```
我方胜率 = 我方战力 / (我方战力 + 对手战力) + 随机波动
```

详细规则：

| 战力差距 | 基础胜率 | 随机波动范围 |
|---------|---------|------------|
| 我方 > 对方 20%以上 | 75%~85% | ±5% |
| 我方 > 对方 10%~20% | 65%~75% | ±8% |
| 双方差距 < 10% | 45%~55% | ±12% |
| 对方 > 我方 10%~20% | 25%~35% | ±8% |
| 对方 > 我方 20%以上 | 15%~25% | ±5% |

> 即使战力碾压，也有约 15%~20% 概率翻车（"以弱胜强"），保持游戏趣味性

### 9.3 快速战斗结果生成

```javascript
function quickBattle(myPower, enemyPower) {
    const ratio = myPower / (myPower + enemyPower);
    const noise = (Math.random() - 0.5) * 0.16;  // ±8%基础波动
    const winProb = Math.max(0.1, Math.min(0.9, ratio + noise));
    const won = Math.random() < winProb;
    
    // 生成比分（Bo5）
    const winnerScore = 3;
    const loserScore = Math.floor(Math.random() * 3);  // 0, 1, 或 2
    
    // 生成击杀数（与战力差相关）
    const diff = Math.abs(myPower - enemyPower) / Math.max(myPower, enemyPower);
    const baseKills = 10 + Math.floor(Math.random() * 8);
    const myKills = won 
        ? baseKills + Math.floor(diff * 10) 
        : baseKills - Math.floor(diff * 5);
    const enemyKills = won 
        ? baseKills - Math.floor(diff * 5) 
        : baseKills + Math.floor(diff * 10);
    
    return {
        won,
        score: won ? [winnerScore, loserScore] : [loserScore, winnerScore],
        kills: { my: Math.max(3, myKills), enemy: Math.max(3, enemyKills) },
        mvp: pickMVP(won),           // 随机选一位表现最好的选手
        highlights: generateHighlights(),  // 3~5条战斗摘要文本
    };
}
```

### 9.6 入口与触发

| 入口位置 | 说明 |
|---------|------|
| **赛事日历·当前比赛** | "进入比赛"按钮旁新增 "⚡ 快速战斗" 按钮 |

快速战斗后：
- 同样触发 `SeasonSystem.recordPlayerMatch()`，更新赛季积分
- 同样触发 `simulateOtherMatches()`，推进同轮其他比赛
- 产出经验、金币等奖励（与正常比赛相同，但无策略加成奖励）
- 返回赛事日历

### 9.7 快速战斗 vs 详细战斗对比

| 对比项 | 详细战斗 | 快速战斗 |
|--------|---------|---------|
| 时间 | 3~10分钟 | 5~8秒 |
| BP环节 | 完整BP | 跳过 |
| 策略选择 | 5个节点手动选择 | 无 |
| 结果决定 | 策略博弈+选手属性+随机 | 战力对比+随机波动 |
| 奖励 | 基础奖励+策略加成(S/A/B评价) | 仅基础奖励 |
| 经验 | 较多（含策略奖励） | 标准量 |
| 碎片 | 焦点赛固定产出 | 焦点赛扫荡产出 |
| 适用场景 | 重要比赛、享受过程 | 赶进度、重复扫荡 |

> 详细战斗给予**额外20%经验和金币奖励**，激励玩家在关键比赛选择详细模式

## 十四、与现有系统的衔接

| 现有系统 | 衔接方式 |
|---------|---------|
| **BP系统** | 选手战力由 rating（动态）决定，高星/高级选手在BP推荐中权重更高；BP顶部实时显示双方战力对比 |
| **比赛系统（详细）** | MatchEngine 读取选手实时 stats（含等级/星级加成），影响各节点策略效果；战力作为参考但不完全决定胜负 |
| **比赛系统（快速）** | QuickBattle 根据双方团队战力计算胜率，生成结果和播报动画，跳过BP和策略环节 |
| **赛事日历** | 焦点赛可扫荡获取指定选手碎片，赛事奖励产出签约券和经验；对战详情面板展示双方战力对比 |
| **城市探索** | 探索事件可获得训练经验、蓝色签约券、少量碎片 |
| **赛季系统** | 每个赛季阶段结束后：刷新转会市场、更新签约池、发放赛季排名奖励；AI战队按赛季阶段自动提升虚拟养成等级 |
| **选手管理** | 选手卡片和详情页展示个人战力值及构成分解；阵容页展示团队总战力及加成明细 |

## 十五、风险与注意事项

1. **数值膨胀**：选手属性上限 99，需要确保满级满星SSR不会过于碾压，保持赛事博弈性
2. **存档兼容**：老存档（没有 level/star/rarity 字段的选手）需要兼容迁移逻辑
3. **工作量**：Phase 1~2 是最小可玩集，建议先交付再迭代
4. **UI复杂度**：签约大厅需要翻牌动画，可以先用简单版本，后续优化
5. **战力公式平衡**：战力值需要在可感知（数字差异明显）和不失衡（碾压感不会太强）之间取平衡；快速战斗的随机波动需保证"以弱胜强"有合理概率但不过于频繁
6. **快速战斗体验**：播报动画时间不宜过长（否则"快速"就没意义），但也不能太短（缺乏仪式感）；建议5~8秒区间，并提供跳过按钮
7. **AI战力曲线**：对手AI的虚拟养成曲线需要仔细调试，确保玩家在前期有养成优势感，后期有挑战压力

# 比赛系统重构策划案 v3

## 一、当前问题诊断

### 1.1 机制问题
| 回合 | 当前设计 | 问题 |
|------|---------|------|
| 回合1·对线期 | 分配3点祝福到三条路 | 操作太简单，只有加减号，缺少策略深度 |
| 回合2·野区博弈 | 五选一（入侵/暴君/gank三路） | 选完就结束，缺乏后续互动 |
| 回合3·中期团战 | QTE环 + 三选一策略（石头剪刀布） | QTE与MOBA无关，策略只有三种太少 |
| 回合4·宏观策略 | 五选一策略卡（石头剪刀布变体） | 与回合3高度重复，体验雷同 |
| 回合5·终局决战 | 二选一/快速选择题 | 过于简化，有时1秒就结束了 |

### 1.2 体验问题
- **回合之间缺乏连贯性**：前面的选择对后面影响微弱
- **信息不透明**：玩家不清楚自己的决策如何影响结果
- **UI缺乏MOBA感**：没有小地图、没有时间线、没有对战视觉
- **节奏单一**：每个回合都是"看描述→点按钮→看结果"
- **缺少动态事件**：比赛过程太线性，没有意外和惊喜

---

## 二、重构核心理念

### 设计哲学
> **"每一步决策都有信息、有权衡、有后果"**

借鉴《杀戮尖塔》《月圆之夜》的卡牌博弈 + 《足球经理》的战术指挥 + KPL转播视角的呈现方式。

### 三大支柱
1. **战术卡牌系统**：用卡牌表示各种战术指令，每张卡有成本、效果、条件
2. **对抗式博弈**：每个回合都是你和对手同时出牌，揭晓后结算
3. **动态战场地图**：用可视化的MOBA地图展示整场比赛的推进

## 四、战术卡牌系统

### 4.1 卡牌分类

| 卡牌名称 | 费用 | 效果 | 条件/备注 |
|---------|------|------|----------|
| 上路强攻 | 2 | 上路推进+8，可能击杀 | 我方对抗路选手面板越高效果越好 |
| 中路游走 | 2 | 中路推进+6，另一路+4 | 需要打野或中路评分>70 |
| 下路压制 | 2 | 下路推进+8 | 发育路&游走的配合值影响 |
| 全线施压 | 3 | 三路各+3 | 高费但覆盖全面 |
| 打野入侵 | 1 | 掠夺对方资源，经济+500 | 打野评分影响成功率 |

| 反蹲守株 | 2 | 若对手进攻该路，反杀+推进 | 读对手的高收益牌 |

| 偷水晶 | 4 | 直接判定胜负（成功率基于差值） | 后期且有兵线优势 |
| 教练暂停 | 0 | 窥视对手全部手牌 + 费用+1 | 每场限1次 |

### 4.2 卡牌机制

- **行动力（费用）**：每轮有 **5点行动力**，出牌消耗费用
- **手牌上限**：最多持有 **6张手牌**
- **每轮抽牌**：回合开始抽 **2张牌**（特定效果可增减）
- **牌库构成**：基于BP阶段选择的英雄和选手数据动态生成
  - 对抗路英雄强 → 更多上路进攻牌
  - 打野评分高 → 更多入侵/gank牌
  - 配合度高 → 更多战术牌
- **弃牌**：可以弃掉不需要的手牌（不消耗费用）

### 4.3 BP如何影响卡牌

| BP因素 | 卡牌影响 |
|--------|---------|
| 阵容配合度高 | 战术牌概率提升，初始手牌+1 |
| 有克制对方英雄 | 对应路线进攻牌效果+20% |
| 被对方克制 | 对应路线防御牌效果-10% |
| 选了T0英雄 | 对应路线可能获得「英雄特技」卡 |
| 对方选了全进攻体系 | 防守反击类牌效果提升 |

## 五、对抗式博弈（每轮结算）

### 5.1 同时出牌制

每轮流程：
1. **抽牌阶段**：自动抽2张，展示新牌
2. **部署阶段**：玩家将卡牌拖放到「指令区」（可出1~3张，受行动力限制）
3. **对手部署**：AI根据局势和战队风格出牌（牌背显示，数量可见）
4. **揭晓阶段**：双方同时翻牌，逐张动画结算
5. **地图更新**：推进/防御/击杀/经济的可视化变化
6. **事件触发**：特定轮次或条件触发随机/固定事件

### 5.2 结算规则

- **同路对抗**：攻击牌 vs 防御牌
  - 攻击力 > 防御力 → 推进成功，推塔/击杀
  - 攻击力 ≤ 防御力 → 进攻被挡
  - 反蹲 vs 攻击 → 防守方反杀
- **不同路线**：如果一路进攻未被防御 → 直接推进
- **经济影响**：经济差每1000转化为全局+1效果
- **选手状态**：高评分选手的牌效果有额外加成

### 5.3 AI出牌逻辑

AI不是随机出牌，而是有性格和策略倾向：

| 战队风格 | 出牌倾向 |
|---------|---------|
| 进攻型（如AG） | 优先出进攻牌，喜欢四人抱团 |
| 运营型（如Hero） | 偏好运营发育+视野布控 |
| 团战型（如TTG） | 频繁使用开团指令 |
| 均衡型（如WB） | 根据局势灵活调整 |
| 防守反击型（如WE） | 前期防守，后期爆发 |

## 六、动态战场地图

### 6.1 地图要素

用简化的MOBA地图呈现（SVG绘制），包含：

- **三条兵线**：上路/中路/下路，各有推进进度条
- **防御塔**：每路2座外塔+1座高地塔，血量可视
- **野区**：暴君/主宰区域，显示刷新倒计时
- **英雄小图标**：5v5英雄头像分布在地图上，根据出牌动态移动
- **战争迷雾**：部分区域雾化，使用「视野布控」可揭露

### 6.2 动态效果

- 出进攻牌 → 对应路线英雄向前推进动画
- 塔被推掉 → 塔图标碎裂动画 + 音效
- 击杀发生 → 被击杀英雄灰色化 + 金币飘字
- 暴君/主宰被击杀 → 全图光效 + buff图标

### 6.3 推进系统

每条路有推进度 0~100：
- 0~30：敌方区域（我方优势）
- 30~50：河道争夺区
- 50~70：我方区域前线
- 70~100：我方高地

推进度影响：
- 到达0 → 敌方外塔被推（-1塔）
- 到达100 → 我方高地塔被推
- 三路高地塔全部被推 → 触发「水晶危机」

## 七、大事件系统

### 7.1 固定事件

| 触发条件 | 事件名称 | 玩法 |
|---------|---------|------|
| 第2轮结束 | 河道之争 | 小型博弈：出1张牌争夺河道资源（经济+视野） |
| 第5轮 | 暴君争夺 | 中型博弈：投入2张牌争夺暴君，双方翻牌，大的赢 |
| 第8轮 | 主宰之战 | 大型博弈：投入3张牌，最终效果决定主宰归属 |
| 势能差>20 | 逆风事件 | 落后方获得1张「绝境翻盘」牌 |

### 7.2 随机事件（每轮10%概率）

| 事件 | 效果 |
|------|------|
| 选手超神 | 随机一名己方选手本轮牌效果翻倍 |
| 操作失误 | 随机一名选手本轮牌效果减半 |
| 团战爆发 | 本轮强制触发团战子博弈 |
| 经济暴涨 | 双方经济各+800 |
| 兵线崩盘 | 某路推进度直接变化±10 |

## 八、团战子博弈（开团指令触发）

当任一方打出「开团指令」牌时，进入团战子博弈：

### 流程
1. **选择团战站位**：5个英雄拖放到前排/中排/后排（3秒限时）
2. **选择核心策略**：
   - 🗡️ 集火后排（适合刺客/战士多）
   - 🛡️ 保护C位（适合射手/辅助强）
   - ✂️ 切割战场（适合法师/坦克多）
   - 🔄 拉扯消耗（适合远程多）
3. **选择击杀目标**：点击想集火的敌方英雄
4. **动画结算**：展示团战过程（英雄冲锋/技能释放的简化动画）
5. **结果**：击杀数、团战胜负、势能变化

### 团战计算
- 基础实力 = Σ 各位选手对应数值
- 策略克制 = 有克制关系时效果 ×1.5
- 站位加成 = 前排坦克/战士+合理分配=额外10%
- 英雄克制 = BP阶段的英雄互相克制生效
- 随机因素 = ±10%

## 九、决胜轮设计

进入决胜轮的条件：完成9轮常规对抗后

### 9.1 顺风（势能≥60）
- 选择终结方式：「强攻水晶」（快速但有风险）or「稳扎稳打」（多1轮但更稳）
- 强攻：60%直接胜利 + 30%被翻盘1轮 + 10%翻盘
- 稳扎稳打：85%胜利

### 9.2 均势（40<势能<60）
- 触发「最终团战」—— 一场完整的团战子博弈
- 结果直接决定胜负

### 9.3 逆风（势能≤40）
- 获得「奇迹」手牌3张（绝境翻盘、偷水晶、完美团战）
- 必须在1轮内翻盘，否则判负
- 成功率30~40%，增加戏剧性

## 十、UI/美术规格

### 10.1 视觉风格
- **主色调**：深蓝底 + 金色点缀（KPL风格）
- **卡牌风格**：渐变底色 + 英雄剪影 + 发光边框
  - 攻击牌：红色渐变
  - 防御牌：蓝色渐变
  - 战术牌：金色渐变
  - 绝技牌：紫色渐变 + 发光粒子
- **地图风格**：简化3D俯视风格，像KPL转播地图
- **动画**：卡牌出牌抛飞、翻牌旋转、推进光线、击杀特效

### 10.2 音效设计
- 抽牌：清脆的翻牌声
- 出牌：卡牌滑入的"嗖"声
- 揭晓：鼓声+翻牌声
- 攻击成功：冲击音效
- 防御成功：护盾音效
- 团战：激烈BGM切换
- 推塔：塔碎裂声
- 击杀：KPL风格"First Blood!"/"Double Kill!"语音

### 10.3 交互方式
- **拖放出牌**：从手牌区拖到指令区
- **点击出牌**：点击卡牌→点击指令槽（移动端友好）
- **长按预览**：长按卡牌查看详细效果
- **双击弃牌**：双击丢弃不需要的牌

## 十一、数值平衡

### 11.1 每轮权重
| 轮次 | 阶段 | 对最终势能的影响上限 |
|------|------|------|
| 1~3 | 前期 | 约30%（每轮±5势能） |
| 4~6 | 中期 | 约35%（每轮±6势能）|
| 7~9 | 后期 | 约25%（每轮±5势能）|
| 决胜 | 终局 | 约10%（基于累积判定）|

### 11.2 经济系统
- 每轮基础收入：500金
- 击杀收益：300金/个
- 推塔收益：500金/座
- 暴君/主宰：800/1200金
- 经济差每2000 → 全局效果+1

### 11.3 胜负判定
- 势能≥65 → 判定胜利（优势过大直接结束）
- 势能≤35 → 判定失败
- 三路高地塔全推 → 直接胜/负
- 9轮后走决胜轮

## 十二、与现有系统的关系

### 保留
- **BP阶段**：完全保留，BP结果影响牌库构成
- **势能条**：保留并增强可视化
- **AI教练**：在卡牌阶段持续提供建议（推荐出什么牌）
- **弹幕/解说**：保留，根据出牌和结算动态生成
- **赛季系统**：比赛结果正常回流到赛季

### 替换
- 回合1~5的所有现有UI和逻辑 → 替换为新的卡牌博弈系统
- QTE环 → 删除（与MOBA无关）
- 快速选择题 → 删除（太简化）

### 新增
- 战术卡牌数据层
- MOBA地图组件
- 卡牌渲染组件
- 拖放交互系统
- 团战子博弈模块
- 大事件系统

# 选手英雄数据采集 & 分级排名计划

## 一、目标

1. 为每位选手建立英雄使用数据库（场次、胜率、KDA）
2. 在BP确认栏中展示"选手使用该英雄的KPL数据"
3. 将所有选手分为 SRR / SR / R 三档

## 二、数据来源分析

| 来源 | 可行性 | 备注 |
|------|--------|------|
| kpl.qq.com | ❌ SPA动态渲染，无公开API | 需Playwright |
| wanplus.cn/kog/ranking | ⚠️ 有选手页面但数据动态加载 | 可尝试抓包 |
| pvp.qq.com/matchdata | ⚠️ 有英雄KPL数据页面 | 旧数据格式 |
| 手动整理+合理模拟 | ✅ 基于现有rating+英雄池生成 | 立即可用 |

**方案选择**：先基于现有选手 `rating`、`PLAYER_HERO_POOLS`、`heroMeta` 合理生成模拟KPL赛季数据，后续可接入真实数据覆盖。

## 三、实施步骤

### Step 1: 创建 `playerStats.js` — 选手英雄使用数据

数据结构：
```javascript
export const PLAYER_STATS = {
    '暖阳': {
        tier: 'SRR',          // 选手分级
        seasonGames: 45,       // 赛季总场次
        seasonWinRate: 62.2,   // 赛季总胜率%
        kda: 5.8,              // 赛季KDA
        mvpCount: 12,          // MVP次数
        heroStats: {
            '镜': { games: 15, wins: 11, kda: 7.2 },
            '裴擒虎': { games: 10, wins: 6, kda: 5.1 },
            '澜': { games: 8, wins: 6, kda: 6.5 },
            '曜': { games: 5, wins: 3, kda: 4.8 },
            '赵云': { games: 4, wins: 3, kda: 5.0 },
            '云中君': { games: 3, wins: 2, kda: 4.5 },
        },
    },
    // ...所有首发选手
};
```

### Step 2: 选手分级系统

基于选手 `rating` + 赛季表现计算：

| 等级 | 条件 | 描述 | 预计人数 |
|------|------|------|----------|
| SRR | rating ≥ 88 且 winRate ≥ 55% | 联赛顶级选手 | ~15人 |
| SR | rating ≥ 80 且 winRate ≥ 50% | 联赛一线选手 | ~30人 |
| R | 其他首发 | 联赛常规选手 | ~45人 |

分级标签在BP确认栏和选手标签页中展示。

### Step 3: 确认栏集成

点击英雄后，确认栏增加：
- `[选手名] 使用 [英雄] 近期数据: 15场 73%胜率 KDA 7.2`
- 如果该选手未使用过此英雄，显示：`[选手名] 尚未在KPL使用过此英雄`

### Step 4: (后续) 真实数据采集

使用 Playwright 从 wanplus.cn 采集真实数据覆盖模拟数据。

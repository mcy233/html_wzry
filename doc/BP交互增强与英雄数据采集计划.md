# BP交互增强 & 英雄数据采集计划

## 一、BP交互增强（立即实施）

### 1.1 选手标签页交互

参考王者荣耀正式比赛的BP界面：

**禁用阶段**：
- 对手（红方）5名选手头像高亮可点击
- 点击某个选手 → 切换到该选手分路的英雄分类页
- 英雄卡片上标注该选手的"常用英雄"（`PLAYER_HERO_POOLS`已有数据）
- 英雄卡片上标注该分路的"强势英雄"（需`heroMeta`数据）

**选用阶段**：
- 我方（蓝方）5名选手头像高亮可点击
- 点击某个选手 → 切换到该选手对应分路
- 显示该选手常用英雄 + 分路强势英雄 + 克制对面已选英雄

### 1.2 选手信息增强显示

每个选手标签显示：
- 头像
- 昵称
- 分路名称（对抗路/打野/中路/发育路/游走）
- 战力值（综合评分）

## 二、英雄数据采集计划（分阶段实施）

### 2.1 数据来源

| 来源 | 数据类型 | 获取方式 |
|------|---------|---------|
| `tianyuanzhiyi.com/heroes` | 英雄胜率、出场率、禁用率、热度 | 浏览器抓包JSON API / Playwright |
| `tianyuanzhiyi.com/hero/{id}` | 单英雄克制、搭配、对线胜率 | 浏览器抓包JSON API / Playwright |
| `pvp.qq.com` 官方API | 英雄ID-名称映射、分路归属 | 直接HTTP请求 |
| 已有 `heroes.js` | 基础英雄列表(130个)、COUNTER_DATA、PLAYER_HERO_POOLS | 本地已有 |

### 2.2 需要采集的数据结构

```javascript
// heroMeta.js — 英雄元数据（版本强度）
export const HERO_META = {
    '镜': {
        tier: 'S',        // S/A/B/C 梯度
        winRate: 52.3,    // 巅峰千强胜率%
        pickRate: 18.5,   // 出场率%
        banRate: 45.2,    // 禁用率%
        hotLanes: ['打野', '对抗路'], // 常用分路
    },
    // ...130个英雄
};

// heroRelations.js — 英雄克制/搭配关系
export const HERO_RELATIONS = {
    '镜': {
        counters: ['后羿', '鲁班七号', '安琪拉'],      // 克制
        counteredBy: ['东皇太一', '张良', '张飞'],       // 被克制
        bestWith: ['蔡文姬', '太乙真人', '大乔'],        // 最佳搭配
        laneWinRates: {                                // 对线胜率
            '赵云': 54.2, '裴擒虎': 48.1, '云中君': 51.0,
        },
    },
    // ...
};
```

### 2.3 采集实施步骤

| 阶段 | 内容 | 工作量 | 方式 |
|------|------|--------|------|
| P0 | 英雄梯度（S/A/B/C）+ 分路归属 | 2h | 人工整理或GPT批量生成 |
| P1 | 130个英雄的克制/搭配关系扩充 | 4h | 基于游戏知识批量生成 |
| P2 | 天元之弈数据抓取脚本 | 3h | Playwright自动化（见下方详细方案） |
| P3 | 选手常用英雄池扩充（所有18支战队） | 3h | 参考wanplus.cn数据 |

### 2.4 P2 详细方案：天元之弈数据采集

天元之弈（tianyuanzhiyi.com）是 SPA 应用，数据通过前端 JS 动态渲染，无公开 REST API。

**采集方案：Playwright 浏览器自动化**

```
步骤1：安装环境
  pip install playwright
  playwright install chromium

步骤2：采集英雄总览数据（胜率/出场率/禁用率）
  - 打开 https://tianyuanzhiyi.com/heroes
  - 选择"巅峰千强"模式
  - 等待表格加载完成
  - 提取所有行数据
  - 输出为 JSON 文件

步骤3：采集单英雄详情（克制/搭配/对线胜率）
  - 遍历130个英雄 ID
  - 打开 https://tianyuanzhiyi.com/hero/{heroId}
  - 等待数据加载
  - 提取克制、搭配、对线胜率数据
  - 注意：需添加合理延迟避免被限流（建议间隔2~3秒）

步骤4：数据整合
  - 将采集的 JSON 数据与 heroMeta.js 格式对齐
  - 生成更新后的 heroMeta.js 和 heroRelations.js
```

**替代方案（更简单）**：
- 手动在浏览器打开天元之弈 → F12 → Network → 找到实际的 XHR 请求URL
- 如果能找到 API 端点，直接用 fetch/axios 请求即可，无需 Playwright
- 已知天元之弈前端使用的技术栈可能有不同的API路径，建议先在浏览器中抓包确认

## 三、游戏中的数据使用方式

### BP阶段
- 英雄卡片角标：`S`/`A`/`B` 梯度标记
- 选手常用标记：该选手常用的英雄标 "⭐常用"
- 强势标记：当前版本高胜率英雄标 "🔥强势"
- 克制标记：克制对面已选英雄标 "⚔️克xxx"
- AI教练引用这些数据给出更精准的推荐

### 训练/阵容界面
- 显示选手的英雄池宽度
- 根据英雄梯度推荐练习方向

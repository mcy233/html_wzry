# KPL比赛 - 卡牌策略系统策划案（简化版）

> 本策划案是对现有比赛系统的补充，将卡牌策略融入现有的5回合比赛流程中

---

## 一、设计目标

基于现有代码结构，将卡牌策略系统**轻量级**地嵌入到当前比赛流程中：

- **不改动**现有的5回合结构（对线期→野区博弈→中期团战→宏观策略→终局决战）
- **增强**回合3（中期团战）和回合4（宏观策略）的策略深度
- **复用**现有的势能条、解说系统、AI教练等组件

---

## 二、卡牌系统概述

### 2.1 卡牌获取时机

在**BP阶段结束后**，根据阵容特点自动生成初始手牌：

```
BP结束 → 系统分析阵容 → 生成5张初始手牌 → 比赛开始
```

### 2.2 卡牌类型（仅3种，简化）

| 类型 | 颜色 | 用途 |
|------|------|------|
| **战术卡** | 金色 | 在回合3（团战）使用，影响团战结果 |
| **策略卡** | 蓝色 | 在回合4（宏观）使用，影响势能走向 |
| **绝技卡** | 紫色 | 任意回合可使用，强力但稀有 |

### 2.3 手牌机制

- **手牌上限**: 5张
- **每回合**: 自动抽1张新牌
- **使用时机**: 
  - 战术卡 → 仅在回合3可用
  - 策略卡 → 仅在回合4可用
  - 绝技卡 → 任意回合可用

---

## 三、具体卡牌设计

### 3.1 战术卡（团战阶段使用）

在回合3「中期团战」时，玩家可选择打出1张战术卡增强团战效果：

| 卡牌名称 | 效果 | 触发条件 |
|----------|------|----------|
| **完美开团** | 团战胜率+25% | 打野评分≥80 |
| **保护后排** | 若对方选"集火后排"，反制成功 | 辅助评分≥80 |
| **侧翼包抄** | 团战胜率+20%，击杀+1 | 边路评分≥80 |
| **技能连招** | 团战胜率+15%，势能额外+5 | 中路评分≥80 |
| **射手接管** | 后期团战胜率+30% | 发育路评分≥80且比赛时间>15分钟 |

**与现有系统的结合**:
- 保留现有的QTE环（时机点击）
- 保留现有的三选一策略（集火/保护/切割）
- **新增**: 在策略选择前，可选择是否打出战术卡

### 3.2 策略卡（宏观阶段使用）

在回合4「宏观策略」时，替代现有的五选一，改为**卡牌对抗**：

| 卡牌名称 | 效果 | 克制关系 |
|----------|------|----------|
| **强开主宰** | 势能+12，高风险 | 被「视野布控」克制 |
| **四一分推** | 势能+8，稳定收益 | 被「五人抱团」克制 |
| **五人抱团** | 势能+10，团战优势 | 被「绕后偷袭」克制 |
| **绕后偷袭** | 势能+12，高风险 | 被「强开主宰」克制 |
| **视野布控** | 势能+6，下回合抽牌+1 | 被「四一分推」克制 |

**与现有系统的结合**:
- 玩家从手牌中选择1张打出
- AI同时出牌（牌背显示）
- 翻牌后根据克制关系结算
- 保留现有的势能条动画

### 3.3 绝技卡（任意回合使用）

稀有卡牌，整场比赛只能获得1-2张：

| 卡牌名称 | 效果 | 获取方式 |
|----------|------|----------|
| **选手觉醒** | 本回合所有属性+30% | 选手状态极佳时概率获得 |
| **教练暂停** | 窥视AI下回合出牌 | 每场比赛限1次 |
| **绝境翻盘** | 势能+15（仅在劣势时可用） | 势能<40时自动获得 |
| **完美运营** | 直接获得本回合最大收益 | 连胜3场后获得 |

## 四、卡牌生成规则

### 4.1 基于BP阵容生成

```javascript
// 伪代码示例
function generateInitialCards(myPicks, myStarters) {
    const cards = [];
    
    // 根据英雄类型添加战术卡
    if (hasHeroType(myPicks, '开团型')) cards.push('完美开团');
    if (hasHeroType(myPicks, '保护型')) cards.push('保护后排');
    if (hasHeroType(myPicks, '刺客型')) cards.push('侧翼包抄');
    
    // 根据选手评分添加战术卡
    myStarters.forEach(player => {
        if (player.rating >= 85) {
            cards.push(getPlayerSignatureCard(player.role));
        }
    });
    
    // 填充基础策略卡到5张
    while (cards.length < 5) {
        cards.push(getRandomStrategyCard());
    }
    
    return cards.slice(0, 5);
}
```

### 4.2 卡牌与阵容的联动

| 阵容特点 | 额外获得卡牌 |
|----------|--------------|
| 多开团英雄（廉颇、牛魔等） | 完美开团 |
| 多保护英雄（张飞、太乙等） | 保护后排 |
| 高机动阵容（韩信、李白等） | 侧翼包抄 |
| 高输出阵容（后羿、妲己等） | 射手接管 |
| 高配合度阵容（>80） | 技能连招 |

## 五、UI界面设计

### 5.1 手牌区域

在现有比赛界面底部增加手牌栏：

```
┌─────────────────────────────────────────────────────────┐
│                    【手牌区域】                           │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────                     │
│  │完美│ │四一│ │选手│ │视野│ │保护│                     │
│  │开团│ │分推│ │觉醒│ │布控│ │后排│                     │
│  │ 2费│ │ 2费│ │ 0费│ │ 1费│ │ 2费│                     │
│  └────┘ └────┘ └────┘ └────┘ └────┘                     │
│   点击选择卡牌（当前可用卡牌高亮显示）                      │
└─────────────────────────────────────────────────────────┘
```

### 5.2 卡牌使用提示

- **可用时**: 卡牌边框发光，可点击
- **不可用时**: 卡牌变灰，显示"本回合不可用"
- **选中时**: 卡牌上浮，显示使用确认按钮

### 5.3 与现有UI的融合

- 保留现有的势能条、解说文字、AI教练面板
- 手牌区域放在AI教练面板下方
- 卡牌打出时有简单的飞入动画

## 七、数值平衡

### 7.1 卡牌效果强度

- 战术卡效果: 相当于选手评分+10~15的效果
- 策略卡效果: 势能变化6~12点
- 绝技卡效果: 相当于改变单回合结果

### 7.2 与现有系统的平衡

- 卡牌效果**叠加**在现有计算之上，不替代
- 不使用卡牌也能正常游戏，使用卡牌获得优势
- AI也会使用卡牌（简化版），保持对抗性

## 九、与完整版卡牌游戏的区别

| 维度 | 本简化版 | 完整版卡牌游戏 |
|------|----------|----------------|
| **卡牌数量** | 15张左右 | 100+张 |
| **卡组构筑** | 无，系统生成 | 玩家自行组卡 |
| **使用时机** | 固定回合 | 任意时机 |
| **复杂度** | 低，5分钟上手 | 高，需要学习 |
| **与现有系统关系** | 增强现有系统 | 独立游戏 |
